## Thank you for your interest in contributing to Instancio!

Contributions can take many forms, all of which are greatly appreciated.

### :computer: Contributing code

Pull requests for bug fixes are greatly appreciated!
If you'd like to add a new feature, please create an issue first
so we can discuss the proposal before implementation. 

To build the project, the following pages should get you started:

- https://www.instancio.org/getting-started/
- https://www.instancio.org/building/

When writing code, consider the following:

- Import the [format config](https://github.com/instancio/instancio/blob/main/ide/instancio-formatter.xml)
  for IntelliJ.
- Follow existing naming conventions.
- Include only the necessary changes for the PR.
- All public APIs should have a javadoc and a `@since` tag, including `package-info.java` for packages.
  - Javadocs should include examples, if applicable.
  - Do not include `@author` tags. Your username will appear on the Contributors page on GitHub after merge.
- Tests should cover all relevant conditional branches whenever possible.
- Squash all your commits into one and rebase your changes onto `main`. Do not merge `main` into your feature branch.
- Create a descriptive commit message.

#### Using AI

We are not against AI tools. However, you are responsible for every line you submit.
Treat AI-generated output as a starting point rather than a finished solution.
Review it thoroughly, remove anything unnecessary, and make sure it follows
the project's conventions and is properly tested.

PRs consisting of unreviewed AI output will be closed.

### :beetle: Bug reports

If you come across a bug or a documentation error, please raise an issue.
For bugs, a minimal reproducible sample would be very helpful for troubleshooting.

### :question: Questions and feedback

Feedback is also very welcome, whether positive or negative.
If you have a question or feedback, please create a post in
[Discussions](https://github.com/instancio/instancio/discussions).
